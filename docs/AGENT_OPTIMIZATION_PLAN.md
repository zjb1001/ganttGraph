# 甘特图 Agent - 自动迭代优化方案
# 基于 learn-claude-code 思想
# 阶段1: MVP (最小可用版本)

## 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                    GanttAgent (极简版)                      │
├─────────────────────────────────────────────────────────────┤
│  1. 基础循环 (Agent Loop)                                    │
│     用户输入 → 理解意图 → 选择工具 → 执行 → 返回结果         │
│                                                              │
│  2. 核心能力 (4个工具)                                       │
│     - read_task: 读取任务                                    │
│     - create_task: 创建任务                                  │
│     - update_task: 更新任务                                  │
│     - export_gantt: 导出甘特图                               │
│                                                              │
│  3. 状态管理                                                 │
│     - 当前项目上下文                                         │
│     - 任务列表缓存                                           │
│     - 用户会话历史 (最近5轮)                                 │
└─────────────────────────────────────────────────────────────┘
```

## 迭代路线图

### 迭代 1: 基础对话 (今晚)
- [ ] 实现基础 Agent Loop
- [ ] 4个核心工具
- [ ] 简单意图识别

### 迭代 2: 任务规划 (明晚)
- [ ] 添加任务依赖分析
- [ ] 自动排期功能
- [ ] 关键路径计算

### 迭代 3: 上下文管理 (周末)
- [ ] 长对话上下文压缩
- [ ] 项目状态持久化
- [ ] 历史记录查询

### 迭代 4: 智能增强 (下周)
- [ ] 风险评估 (基于历史数据)
- [ ] 资源冲突检测
- [ ] 建议生成

### 迭代 5+: 按需添加
- [ ] 子代理 (复杂分析任务隔离)
- [ ] 技能系统 (特定领域知识)
- [ ] 多项目协作

## 反模式检查清单

- [ ] 没有超过5个核心工具？
- [ ] 没有复杂预设工作流？
- [ ] 知识按需加载而非前置？
- [ ] 让模型决定而非硬编码？

## 成功标准

**迭代1完成标志**:
- 用户可以说"创建一个任务，叫设计评审，3月10日开始，持续3天"
- Agent能正确解析并执行
- 无需其他复杂功能

**迭代2完成标志**:
- 能自动处理任务依赖
- 能识别关键路径
- 能给出延期预警

---

*优化原则: 先能用，再好用，最后自动化*
